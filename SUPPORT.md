# Support

Thanks for using or evaluating Zuri Market. Here's how to get help.

## Reporting a Bug

Open an issue using the **Bug Report** template under
`.github/ISSUE_TEMPLATE/`. Include steps to reproduce, expected vs.
actual behavior, and environment details where relevant.

## Requesting a Feature

Open an issue using the **Feature Request** template under
`.github/ISSUE_TEMPLATE/`. Describe the problem you're trying to solve,
not just the solution you have in mind — it helps us evaluate fit with
the roadmap (see [`ROADMAP.md`](./ROADMAP.md)).

## Security Issues

Do **not** open a public issue for security vulnerabilities. Follow the
process in [`SECURITY.md`](./SECURITY.md) instead.

## General Questions

For general questions about the project, direction, or access:

- Use [GitHub Discussions](../../discussions), if enabled for this
  repository
- Or contact the Owner directly — see [`OWNER.md`](./OWNER.md)

## Licensing and Usage Questions

Zuri Market is proprietary software. If your question is about
licensing, permitted use, or a commercial arrangement, see
[`LICENSE`](./LICENSE) and [`TERMS_OF_USE.md`](./TERMS_OF_USE.md) first,
then reach out to the Owner directly for anything not covered there.

## Response Times

This project is maintained by a small, focused team. We aim to
acknowledge issues and questions promptly but response times may vary
depending on severity and current workload.
